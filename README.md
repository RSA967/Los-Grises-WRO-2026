# Los-Grises-WRO-2026

Official repository of Team Los Grises for the **Future Engineers - World Robot O1ympiad 2026**.

The code of our robot is designed using a modular architecture, meaning it is divided into multiple independent modules, each responsible for a specific function within the system. This approach not only improves code organization but also facilitates maintenance, scalability, and understanding. Thanks to this structure, each module can be developed, tested, and optimized individually, reducing the possibility of errors and allowing improvements without affecting the overall system. At the same time, all modules are interconnected, enabling the robot to function as a unified system capable of interacting with its environment in real time.

One of the most important modules is the vision module, in which the HuskyLens sensor is used. This sensor plays a fundamental role, as it functions as the visual system of the robot. By using built-in artificial intelligence, the HuskyLens is capable of recognizing colors, shapes, and previously trained objects. This allows the robot to identify specific elements within its environment, which is essential for autonomous operation. The sensor not only detects objects but also provides detailed information such as their position along the X and Y axes, as well as their dimensions, including width and height. These data are crucial for the robot to accurately interpret its surroundings and make decisions based on precise information.

To ensure optimal performance of the vision system, a calibration process is carried out beforehand. This process includes adjustments in lighting, sensitivity, and color recognition, which improves the accuracy of the sensor under different environmental conditions. Additionally, object training is performed carefully, taking into account possible variations in size, orientation, and lighting. This ensures that the robot can recognize objects even in changing situations, which is especially important in competition environments.

The control module is another essential component of the system. This module is responsible for managing program execution through a continuous loop that allows the robot to constantly monitor its environment. Thanks to this loop, the robot can react in real time to any changes detected by the vision sensor. The control module also includes safety mechanisms, such as a stop button that allows the robot to be immediately halted if necessary. This is particularly useful during testing, where control and safety are critical aspects.

Within this module, logical structures such as conditionals and loops are implemented, enabling the robot to make basic decisions. For example, depending on the position of an object, the robot can execute different actions. This type of logic forms the foundation of autonomous behavior, allowing the robot to respond dynamically to its environment.

The data processing module is responsible for transforming the information obtained from the sensor into useful data for the system. The data captured by the camera are stored in variables, allowing them to be analyzed and used within the program. At the current stage, these data are displayed on a screen for monitoring and debugging purposes. However, in future stages, this information will be used for more complex decision-making, such as object tracking or obstacle avoidance.
This module may also include mathematical calculations that help interpret the data more effectively. For instance, the robot can estimate the distance to an object based on its apparent size or determine whether it needs to adjust its trajectory to align with a target. These functions are essential for developing more advanced behaviors.

Regarding the physical components, the code is designed to work together with the robot’s electromechanical elements, such as motors, controllers, and additional sensors. The motors enable movement, while the controller regulates the power supplied to them, ensuring stable operation. Although the motors are not directly controlled at this stage, the information generated by the vision system will later be used to determine how the robot should move.

   In future improvements, more advanced control algorithms such as proportional control or PID control are planned to be implemented. These will allow smoother and more precise movements, which are especially useful for tasks that require high accuracy, such as following paths or aligning with objects.
The development of the code follows a structured process. First, it is written in the corresponding programming environment. Then, it is carefully reviewed to detect errors and ensure proper functionality. Once validated, the code is compiled into a format that the robot can execute. Afterward, the program is transferred to the robot via a USB cable, enabling autonomous operation.

Another important aspect is code optimization. Efforts are made to reduce execution time and improve resource usage, which is essential for achieving efficient real-time performance. Additionally, potential system failures, such as detection errors or lighting changes, are considered, and solutions are implemented to maintain system stability.

Finally, this system is designed with future development in mind. Its modular structure allows the integration of new technologies, such as more advanced artificial intelligence algorithms, autonomous navigation systems, and improved interaction with the environment. This makes the robot a scalable platform capable of evolving and adapting to new challenges.

In conclusion, the developed code provides a solid foundation for the robot’s operation. It enables environmental perception, data processing, and preparation for autonomous decision-making. Thanks to its modular design, integration with sensors, and focus on continuous improvement, the robot has the potential to developincreasingly complex behaviors, which are essential for success in competitions such as WRO and in the field of robotics in general.
